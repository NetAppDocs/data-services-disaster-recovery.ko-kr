= NetApp Disaster Recovery 작업 모니터링
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ../media/use/


[role="lead"]
모든 NetApp Disaster Recovery 작업을 모니터링하고 진행 상황을 확인할 수 있습니다.



== 채용공고 보기

*필요한 NetApp Console 역할* 이 작업을 수행하려면 슈퍼 관리자, 재해 복구 관리자, 재해 복구 애플리케이션 관리자 또는 재해 복구 뷰어 역할이 필요합니다. link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-disaster-rec-roles.html["NetApp Disaster Recovery 의 사용자 역할 및 권한에 대해 알아보세요."^]. link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html["모든 서비스에 대한 NetApp Console 액세스 역할에 대해 알아보세요."^].

.단계
. 에 로그인하세요 https://console.netapp.com/["NetApp Console"^] .
. NetApp Console 왼쪽 탐색에서 *보호* > *재해 복구*를 선택합니다.
. 메뉴에서 *작업 모니터링*을 선택합니다.
. 운영과 관련된 모든 직무를 탐색하고 타임스탬프와 상태를 검토합니다.
. 특정 직업의 세부 정보를 보려면 해당 행을 선택하세요.
. 정보를 새로 고치려면 *새로 고침*을 선택하세요.




== 작업 취소

진행 중이거나 대기 중인 작업이 중단된 경우 또는 보류 중인 작업의 우선 순위를 지정하기 위해 해당 작업을 취소할 수 있습니다.

*필요한 NetApp Console 역할* 이 작업을 수행하려면 Super admin, Disaster recovery admin, Disaster recovery failover admin 또는 Disaster recovery application admin 역할이 필요합니다. link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-disaster-rec-roles.html["NetApp Disaster Recovery 의 사용자 역할 및 권한에 대해 알아보세요."^]. link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html["모든 서비스에 대한 NetApp Console 액세스 역할에 대해 알아보세요."^].

.단계
. NetApp Console 왼쪽 탐색 모음에서 *보호* > *재해 복구*를 선택합니다.
. 메뉴에서 *작업 모니터링*을 선택합니다.
. 작업 모니터 페이지에서 취소하려는 작업의 ID를 기록해 둡니다.
+
작업은 "진행 중" 또는 "대기 중" 상태여야 합니다.

. 작업 열에서 *작업 취소*를 선택합니다.
